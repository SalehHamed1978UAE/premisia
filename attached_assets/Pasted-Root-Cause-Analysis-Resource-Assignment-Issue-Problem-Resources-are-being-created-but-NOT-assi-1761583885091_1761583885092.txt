Root Cause Analysis - Resource Assignment Issue

  Problem: Resources are being created but NOT assigned to tasks in
  QgenticEPM3.

  Root Cause Found: Data structure mismatch between intelligent planning task
   format and assignment engine expectations.

  The Issue:

  Intelligent Planning generates tasks with:
  {
    name: "Design Database Schema",
    requirements: [
      { skill: "Database Design", quantity: 1, duration: 10 },
      { skill: "SQL", quantity: 1, duration: 10 }
    ]
  }

  Assignment Engine expects:
  {
    name: "Design Database Schema",
    skills: ["Database Design", "SQL"]  // Flat array
  }

  In /server/intelligence/epm-synthesizer.ts line 2083:
  requiredSkills: task.skills || []  // Always returns [] when task has 
  requirements instead!

  Result: Assignment matching fails because tasks have no requiredSkills, so:
  - Skill matching returns 0 points
  - Role matching might give +20 points
  - Very weak matches or NO matches
  - No assignments created!
